.Net (Dot Net)





                       

Ultimamente temos ouvido falar bastante em Dot NET. Mas afinal que danado é esse tal de Dot NET?

Para entendermos o que de fato é o Dot NET, precisamos entender primeiro o conceito de Framework.

Existem varias definições para framework mas a idéia aqui é de que um framework é um conjunto de classes, objetos e metodologias para desenvolvimento. Para ficar mais claro, você lembra daquele comercial? “Existe mil maneiras de se preparar Neston, invente uma!”. Como você pode preparar seu Neston, de várias formas ok? Liquidificador, triturador, batedeira, etc… Que tal definirmos um Framework para nosso Neston? Um Framework para o Neston seguiria o seguinte padrão: Liquidificar- 2 Colheres sopa de Neston- 1 Colher de açucar- 1 Banana- 1/2 Maçã.

Observe que aqui definimos um padrão para fazermos o Neston, o conjunto do liquidificador, frutas, açucar, forma nosso Framework!

Voltando para o mundo virtual um Framework é um conjunto de metodos, padrões e classes que definem e disponibilizam recursos para o desenvolvimento. O Dot NET é isso, um Framework de desenvolvimento. Com ele você não precisa investir seu tempo para fazer determinadas tarefas pois ele já tem um caminho para que esta tarefa seja feita, basta apenas você usar.
